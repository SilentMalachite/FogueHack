## 📝 ドキュメント改善

ドキュメントの改善提案は大歓迎です：

1. **誤字・脱字**: 直接 Pull Request をお送りください
2. **内容の改善**: GitHub Issues で提案してください
3. **新しいドキュメント**: GitHub Discussions で相談してください

### ドキュメント作成ガイドライン

- **読者を意識**: 対象者（初心者/上級者）を明確に
- **構造化**: 見出し、リスト、表を効果的に使用
- **具体例**: コード例、実行例を豊富に提供
- **最新性**: 実装と齟齬がないよう定期更新
- **日本語**: 基本的に日本語で記述（コードコメントも）
